分析一个奇怪的程序

Apr 19, 2017

分析下面的程序，思考：这个程序可以正确返回吗？

assume cs:code
code segment

        mov    ax,4c00h
        int 21h

start:    mov ax,0
    s:    nop
        nop

        mov di,offset s
        mov si,offset s2
        mov ax,cs:[si]
        mov cs:[di],ax

   s0:    jmp short s

   s1:    mov ax,0
        int 21h
        mov ax,0

   s2:    jmp short s1
        nop

code ends
end start

查看CPU指令

##指令执行顺序

  1. ip＝5，执行代码中的 mov ax,0
  2. 按-t 一直执行到0014 mov cs:[di],ax
  3. 执行指令0016 jmp short s (jmp 0008)
  4. 执行指令 jmp 0000
  5. 执行指令0000 mov ax,4c00h
  6. 执行指令0003 int 21h

#结论：程序可以正常退出

分析

mov di,offset s
mov si,offset s2
mov ax,cs:[si]
mov cs:[di],ax

这段代码的作用是将指令 s2 对应的指令当作数据存储 位于指令 s1 对应的地址处

指令s2 对应的机器代码为 EBF6
其中 EB 代表移动， F6 代表位移。

执行完这条指令后，我们查看指令（按-u cs:0） 发现原本指令0C8D:0008 处的90（NOP）成为了 EBF6 (jmp 0000)

转移指令的作用

  1. 从 CS:IP 指向内存单元读取指令，读取的指令进入指令缓存器;
  2. (IP)=(IP)+所读取指令的长度，从而指向下一条指令;
  3. 执行指令。转到 1， 重复这个过程

F6是负值

Not(F6=11110110) + 1 = 1010b = 10

即ip向前移动10个字节

实际上读取指令 s2: jmp short s1 到缓存后

ip = 0022

之后 CPU 从缓存中读取s2指令，跳转到s1：0018处

同样可以得到 ip向前移动10个字节

22h-18h = 0ah = 10

接下来 CPU 执行指令 0016 s0:jmp short s

再跳转到指令s处

同上，CPU 读取指令 “ip向前移动10位” 到缓存中,

由于该指令存放在16位寄存器ax中，长度为2个字节

接着ip指向地址：000A，

接下来cpu从缓存中读取指令并执行，即
0ah － 10 ＝ 0

所以ip＝0，接着执行
mov ax，4c00h; int 21h;

#程序正常退出！！！
